# Utah-Vid-ia — нетехнический обзор

## Одним предложением

**Utah-Vid-ia — это middleware, которое позволяет ПО для ИИ и вычислений работать гибче на разных GPU, с лучшей утилизацией кластера и меньшим числом сбоев из-за глюков памяти.**

## Почему это важно

Вендоры GPU продают чипы **плюс** программную экосистему. Привязка к CUDA от NVIDIA — самый известный пример: команды вкладывают годы в код только под CUDA, а потом с трудом используют AMD, Intel или более дешёвое железо без полной переписки.

Utah-Vid-ia находится **между вашим приложением и железом**. Оно не заменяет PyTorch или вашу модель ИИ — оно добавляет слой переносимости и оркестрации.

## Какие проблемы решает

| Проблема | Подход Utah-Vid-ia |
|----------|-------------------|
| Привязка к вендору | Ghost kernel + переносимые IR-метаданные |
| Статичные драйверы одного размера | Перекомпиляция векторов в рантайме (BRTR) |
| Неравномерная нагрузка GPU-кластера | Осмотическая маршрутизация на несколько GPU |
| Простой GPU в ожидании CPU | Neural-State Pre-fetching (ZEO) |
| Накладные расходы ECC / тихая порча | Entropy-Shield — исцеление многообразия |
| Изолированные / без-RF связи | Photonic bridge (встроенный прототип) |

## Кому полезно

- **AI-стартапы** — растягивают бюджет на железо на смешанных GPU-флотах
- **Рендер-фермы и лаборатории** — балансируют нагрузку без кастомных планировщиков на каждого вендора
- **Предприятия** — снижают риск переговоров с одним вендором
- **Суверенные / офлайн-развёртывания** — фотонная сеть (экспериментально)

## Чем это *не* является (сегодня)

- Не волшебная кнопка «10× быстрее на любой нагрузке»
- Не полная замена CUDA с первого дня (альфа v0.2.0)
- Не сертифицировано для регулируемого продакшена без вашей собственной валидации

## Как оценить (30 минут)

1. Клонируйте [github.com/utahisnotastate/utahvidia](https://github.com/utahisnotastate/utahvidia)
2. Запустите `utahvidia` для живого демо
3. Запустите `utahvidia bench` на ваших GPU
4. Прочитайте [руководство для CTO](cto.md), если решаете бюджет или стратегию по вендорам
